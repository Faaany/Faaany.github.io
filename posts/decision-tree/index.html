Brief Introduction to Decision Trees

05 Sep 2020
@Fanli

As the name implies, a decision tree is a tree for making decisions. It is especially useful, when you need to make a decide among several options. For example, a bank wants to decide whether it should provide a loan to a specific customer and it has some data about this customer. In this case, the bank could use a decision tree like the one below to to help him make the decision:

There are three major components in a decision tree: decision nodes, leaf nodes and links, which are presented with circles, rectangles and arrows respectively in the graph above.

  • Decision node represents a feature. The decision node at the top of the tree has a special name called root node. In our example, it is the income range of an applicants shown in yellow.
  • The leaf node represents an outcome. It can be categorical and numerical. In our case, it is loanor no loan shown in green.
  • The link between nodes represents a decision rule, e.g. the different income range of an applicant.

A bank clerk could use his years of experiences and domain expertise to draw this kind of decision tree. But there is a machine learning approach to do it, namely using data to train a decision tree.

How to build a decision tree?

Given a training data set with different features and target variable, the first step of building a decision tree is to decide which feature should be used in the root node. The basic idea is that the feature that best classifies the whole training data should be used in the root node. But how do we quantify how well a feature classify the training data? There are two common measurements: Information Gain and Gini Index.

The idea of information gain is that the data should have a lower entropy after being split on a feature. First you need to calculate the entropy before splitting. Then calculate the entropies after the split on all possible features in the data. The feature that have the largest difference on entropy (largest information gain) after splitting will be used as the root node.

Gini index is a metric to measure how often a randomly chosen element would be incorrectly identified. Therefore, the attribute with lower Gini index after splitting should be preferred. To better understand how gini index is calculated on a concrete example , check out this StatQuest video about Decision Tree.

After the feature of the root node is chosen, all we need to do is to repeat the same procedure on every branch until the decision nodes of each branch is finalised.

Summary

Decision tree is a supervised machine learning algorithm that can be used to solve both regression and classification problems. It breaks down a data set into smaller and smaller subsets while at the same time a decision tree with decision nodes and leaf nodes is incrementally developed. Decision tree can handle both categorical data and numerical data and can tell us the feature importance in the data.
